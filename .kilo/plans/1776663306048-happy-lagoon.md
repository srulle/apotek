# Rencana Implementasi Sistem Log Stok dan Penjualan

## Overview

Implementasi sistem audit trail untuk perubahan stok melalui tabel `log_stok`, serta struktur database penjualan yang konsisten dengan pembelian (tanpa customer data).

## Tujuan

- ✅ Mencatat setiap perubahan stok secara otomatis
- ✅ Audit trail lengkap untuk pembelian dan penjualan
- ✅ Struktur database konsisten antara pembelian dan penjualan
- ✅ Tanpa UI/frontend untuk log (hanya backend logging)

## Urutan Implementasi

### 1. **Buat Tabel `log_stok`**

**File:** `database/migrations/2026_04_20_131352_create_log_stok_table.php`
**Tujuan:** Tabel untuk menyimpan history perubahan stok
**Kolom:**

- `id` (primary)
- `obat_id` (FK ke obat)
- `nomor_batch`, `tanggal_expired`
- `jenis_transaksi` (pembelian/penjualan)
- `jumlah_masuk`, `jumlah_keluar`
- `stok_sebelum`, `stok_sesudah`
- `referensi_id` (FK ke pembelian/penjualan)
- `keterangan`
- `created_at`

### 2. **Buat Model `LogStok`**

**File:** `app/Models/LogStok.php`
**Tujuan:** Model untuk berinteraksi dengan tabel log_stok
**Features:**

- Fillable fields sesuai kolom tabel
- Casts untuk date dan integer fields
- Relationship ke Obat (opsional)

### 3. **Update `Stok` Model**

**File:** `app/Models/Stok.php`
**Tujuan:** Menambahkan `$table = 'stok'` untuk konsistensi nama tabel

### 4. **Integrasikan Log ke `PembelianController`**

**File:** `app/Http/Controllers/PembelianController.php`
**Tujuan:** Catat log setiap kali stok berubah saat pembelian
**Changes:**

- Import model LogStok
- Dalam transaction, setelah update/create stok, catat log
- Log format: jenis_transaksi='pembelian', jumlah_masuk=jumlah_beli, dll

### 5. **Buat Migration Tabel `penjualan`** ✅ COMPLETED

**File:** `database/migrations/2026_04_24_062716_create_penjualan_table.php`
**Tujuan:** Tabel header penjualan (tanpa customer_id)
**Kolom:**

- `id` (ULID primary)
- `nomor_faktur` (unique)
- `tanggal_penjualan` (date)
- `user_id` (FK ke users)
- `total_harga` (decimal, opsional)
- Indexes dan constraints

### 6. **Buat Migration Tabel `penjualan_detail`** ✅ COMPLETED

**File:** `database/migrations/2026_04_24_062752_create_penjualan_detail_table.php`
**Tujuan:** Tabel detail item penjualan
**Kolom:**

- `id` (ULID primary)
- `penjualan_id` (FK ke penjualan)
- `obat_id`, `nomor_batch`, `tanggal_expired`
- `jumlah_jual`, `harga_jual`
- Indexes dan unique constraints

### 7. **Buat Model `Penjualan` dan `PenjualanDetail`** ✅ COMPLETED

**File:** `app/Models/Penjualan.php`, `app/Models/PenjualanDetail.php`
**Tujuan:** Models untuk penjualan tables
**Features:**

- ULID support
- Relationships (penjualan hasMany detail, belongsTo user)
- Fillable dan casts sesuai struktur

### 8. **Buat `PenjualanController`** ✅ COMPLETED

**File:** `app/Http/Controllers/PenjualanController.php`
**Status:** ✅ Controller lengkap dengan method:

- `index()`: Load data obat + stok untuk frontend
- `store()`: Simpan penjualan dengan validasi, stok checking, database transaction
- Error handling: stok tidak cukup, constraint violations, system errors
- Auto-logging ke `log_stok` setiap perubahan stok
- Menggunakan authenticated user (bukan dari frontend)

### 9. **Tambah Route untuk Penjualan** ✅ COMPLETED

**File:** `routes/web.php`
**Status:** ✅ Routes sudah ditambahkan:

- `GET transaksi/penjualan` → `PenjualanController@index` (load data obat + stok)
- `POST transaksi/penjualan` → `PenjualanController@store` (simpan transaksi)

### 10. **Buat Frontend Penjualan** ✅ COMPLETED

**Files:** `resources/js/pages/transaksi/penjualan/PenjualanForm.tsx`
**Status:** ✅ Frontend sudah dibuat dengan fitur:

- Form input tanggal transaksi
- Combobox untuk pilih obat (dengan stok batch)
- Table detail penjualan dengan batch selection
- Generate nomor faktur otomatis
