# Rencana Implementasi Sistem Log Stok dan Penjualan

## Overview

Implementasi sistem audit trail untuk perubahan stok melalui tabel `log_stok`, serta struktur database penjualan yang konsisten dengan pembelian (tanpa customer data).

## Tujuan

- ✅ Mencatat setiap perubahan stok secara otomatis
- ✅ Audit trail lengkap untuk pembelian dan penjualan
- ✅ Struktur database konsisten antara pembelian dan penjualan
- ✅ Tanpa UI/frontend untuk log (hanya backend logging)

## Urutan Implementasi

### 1. **Buat Tabel `log_stok`**

**File:** `database/migrations/2026_04_20_131352_create_log_stok_table.php`
**Tujuan:** Tabel untuk menyimpan history perubahan stok
**Kolom:**

- `id` (primary)
- `obat_id` (FK ke obat)
- `nomor_batch`, `tanggal_expired`
- `jenis_transaksi` (pembelian/penjualan)
- `jumlah_masuk`, `jumlah_keluar`
- `stok_sebelum`, `stok_sesudah`
- `referensi_id` (FK ke pembelian/penjualan)
- `keterangan`
- `created_at`

### 2. **Buat Model `LogStok`**

**File:** `app/Models/LogStok.php`
**Tujuan:** Model untuk berinteraksi dengan tabel log_stok
**Features:**

- Fillable fields sesuai kolom tabel
- Casts untuk date dan integer fields
- Relationship ke Obat (opsional)

### 3. **Update `Stok` Model**

**File:** `app/Models/Stok.php`
**Tujuan:** Menambahkan `$table = 'stok'` untuk konsistensi nama tabel

### 4. **Integrasikan Log ke `PembelianController`**

**File:** `app/Http/Controllers/PembelianController.php`
**Tujuan:** Catat log setiap kali stok berubah saat pembelian
**Changes:**

- Import model LogStok
- Dalam transaction, setelah update/create stok, catat log
- Log format: jenis_transaksi='pembelian', jumlah_masuk=jumlah_beli, dll

### 5. **Buat Migration Tabel `penjualan`**

**File:** `database/migrations/2026_04_21_140000_create_penjualan_table.php`
**Tujuan:** Tabel header penjualan (tanpa customer_id)
**Kolom:**

- `id` (ULID primary)
- `nomor_faktur` (unique)
- `tanggal_penjualan` (date)
- `user_id` (FK ke users)
- `total_harga` (decimal, opsional)
- Indexes dan constraints

### 6. **Buat Migration Tabel `penjualan_detail`**

**File:** `database/migrations/2026_04_21_140100_create_penjualan_detail_table.php`
**Tujuan:** Tabel detail item penjualan
**Kolom:**

- `id` (ULID primary)
- `penjualan_id` (FK ke penjualan)
- `obat_id`, `nomor_batch`, `tanggal_expired`
- `jumlah_jual`, `harga_jual`
- Indexes dan unique constraints

### 7. **Buat Model `Penjualan` dan `PenjualanDetail`**

**File:** `app/Models/Penjualan.php`, `app/Models/PenjualanDetail.php`
**Tujuan:** Models untuk penjualan tables
**Features:**

- ULID support
- Relationships (penjualan hasMany detail, belongsTo user)
- Fillable dan casts sesuai struktur

### 8. **Buat `PenjualanController`**

**File:** `app/Http/Controllers/PenjualanController.php`
**Tujuan:** Controller untuk handle transaksi penjualan
**Features:**

- Method store() dengan validation
- Logic untuk reduce stok dan catat log_stok
- Error handling jika stok tidak mencukupi
- Redirect success message

### 9. **Tambah Route untuk Penjualan**

**File:** `routes/web.php`
**Tujuan:** Route untuk transaksi penjualan
**Routes:**

- `Route::get('transaksi/penjualan', ...)`
- `Route::post('transaksi/penjualan', [PenjualanController::class, 'store'])`

### 10. **Buat Frontend Penjualan (Opsional)**

**Files:**

- `resources/js/pages/transaksi/penjualan/PenjualanForm.tsx`
- `resources/js/pages/transaksi/penjualan/usePenjualanForm.ts`
  **Tujuan:** UI untuk input penjualan
  **Notes:** Mirip dengan PembelianForm tapi untuk penjualan

## Testing & Validation

### 11. **Run Migrations**

```bash
php artisan migrate
```

### 12. **Test Pembelian dengan Log**

- Input transaksi pembelian
- Verify: data tersimpan di pembelian, pembelian_detail, stok, log_stok
- Check log_stok entries untuk jenis_transaksi='pembelian'

### 13. **Test Penjualan dengan Log**

- Input transaksi penjualan
- Verify: stok berkurang, log_stok tercatat dengan jenis_transaksi='penjualan'
- Test error handling jika stok tidak cukup

## Dependencies & Prerequisites

- ✅ Laravel framework sudah terinstall
- ✅ Database MySQL/PostgreSQL
- ✅ Model Stok, Pembelian, PembelianDetail sudah ada
- ✅ Migration tables sudah di-run
- ✅ Frontend React/Inertia sudah setup

## Notes

- Log_stok tidak memiliki UI/frontend - hanya backend logging
- Penjualan tidak menyimpan customer data (sesuai requirement)
- Semua logging terjadi secara otomatis dalam database transaction
- Error dalam transaction akan rollback semua changes termasuk log</content>
